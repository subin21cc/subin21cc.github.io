## Speech Script (100초)

### **1. 문제**

여러분, 건강 상태를 정확히 알고 관리하고 계신가요? 최근 5년간 20~30대의 당뇨병 유병률은 38%, 고혈압은 28%나 증가했습니다. 하지만 기존 건강관리 앱은 번거로운 수동 입력과 획일적인 조언에 그쳐, 바쁜 청년들이 꾸준히 관리하기 어려운 한계가 있습니다.

### **2. 해결 아이디어**

On-Care는 사진 한 장으로 식단을 기록·분석하고, 누적 건강 데이터를 학습한 AI 코치가 개인 맞춤 피드백을 제공합니다. 또한 식단, 운동, 헬스장, 일정 관리까지 하나의 앱에서 통합적으로 해결합니다.

### **3. 기술 / 구현**

식단 인식은 2단계 파이프라인으로 동작합니다. 먼저 YOLOv8이 음식 여부를 판별해 불필요한 API 호출을 차단하고, Gemini Vision API가 음식 종류와 섭취량을 분석해 영양 정보를 자동 기록합니다. AI 코칭은 RAG 기반으로 구현하고 개인 건강 데이터를 Pinecone에 저장하여, 사용자 기록을 반영한 맞춤형 답변을 제공합니다. 시스템은 FastAPI, Flutter 기반으로 구축됩니다.

### **4. MVP / 배포**

현재 Gemini API 연동과 프롬프트 설계를 완료했습니다. MVP에서는 사진 기반 식단 기록, AI 코칭, 운동 루틴 및 캘린더 연동 기능을 제공합니다. 앱은 iOS와 Android 동시 출시를 목표로 개발 중이며, 서비스는 Docker 환경에서 운영할 예정입니다.

### **5. 차별성**

기존 앱들은 기록과 코칭이 분리되거나, 오프라인 운동 환경과 단절되어 있습니다. 저희 서비스, On-Care는 내 누적 기록을 실시간 참조하는 RAG 기반 AI 코치와 헬스장 연동을 하나로 묶어, 기록에서 실제 행동 변화까지 연결합니다.

## Tech Stack: 구현 아키텍처

효율적인 비용 운영과 고도의 개인화를 위해 다음과 같은 기술 스택을 선택했다.

- **Mobile:** `Flutter` (iOS·Android 동시 지원)
- **Backend:** `FastAPI`, `Docker`, `AWS EC2`
- **Vision AI:** `YOLOv8` (필터링) + `Gemini Vision API` (영양 분석)
- **AI Coaching:** `LangChain` + `Pinecone` + `GPT-4o` (RAG 파이프라인)
- **Data:** 공공데이터 식품영양성분 DB 매핑

## 고객 인터뷰

실제 2030 만성질환 위험군 및 환자 3인을 인터뷰한 결과, 공통적인 **Pain Point**를 발견했다.

> *"기록을 해도 내 상태에 맞는 피드백이 없으니 3일 이상 쓰기 힘들어요."*
>
>
> *"식단이 매번 비슷한데 하나하나 검색해서 입력하는 게 너무 고역입니다."*
>

**On-Care의 해답:**

1. **기록 자동화:** 사진 한 장으로 끝내는 간편함.
2. **맥락 있는 피드백:** "이번 주 탄수화물이 많으니 저녁은 샐러드를 드세요"와 같은 실질적 가이드.
3. **오프라인 연결:** 앱 내 데이터를 트레이너에게 공유하여 상담 효율 극대화.

## 핵심 Q&A

**Q: 왜 2030 세대의 만성질환 위험군을 타겟으로 했나요?**

**A:** 최근 5년간 2030세대의 당뇨병(38%)과 고혈압(28%) 유병률이 급증하고 있습니다. 하지만 이들은 불규칙한 생활 패턴으로 인해 기존 헬스케어 앱의 수동 기록 방식에 쉽게 피로를 느끼고 이탈하는 경향이 있어, 이를 자동화로 해결하고자 했습니다.

**Q: 기존 헬스케어 앱(삼성헬스, 인바디 등)과 무엇이 다른가요?**

**A:**기존 앱이 체중 감량이나 체성분 측정에 집중한다면, On-Care는**고혈압·당뇨 관리**에 특화되어 있습니다. 칼로리 계산을 넘어 나트륨, GI 지수 기반 분석을 제공하며, 기록이 단순 저장을 넘어 실질적인 오프라인 행동(헬스장/트레이너)으로 연결되는 구조를 가집니다.

**Q: YOLOv8과 Gemini Vision API를 함께 사용하는 이유는 무엇인가요?**

**A:** **비용 효율성**과 **분석 정밀도**를 동시에 잡기 위함입니다. 가벼운 YOLOv8이 음식 이미지 여부를 1차로 판별하여 불필요한 API 호출을 막고 , 음식인 경우에만 고성능 Gemini Vision API를 호출해 정밀한 영양 분석을 수행함으로써 비용을 획기적으로 절감합니다.

**Q: RAG(검색 증강 생성) 파이프라인에서 Pinecone을 선택한 이유는 무엇인가요?**

**A:** 완전 관리형 서비스로 인프라 운영 부담을 줄이면서도 밀리초 단위의 고속 검색을 지원하기 때문입니다. 또한 사용자별 메타데이터 필터링을 통해 개인의 식단·운동 이력에 기반한 **초개인화된 AI 코칭**을 구현하는 데 적합합니다.

**Q: 사진 한 장으로 분석하는 영양 정보의 정확도를 어떻게 보장하나요?**

**A:** AI의 추정치를 그대로 쓰지 않고, 인식된 음식명을 **공공데이터 식품영양성분 DB**와 매핑하여 검증된 수치를 제공합니다. 또한 사용자가 직접 수정할 수 있는 UI를 제공해 데이터 무결성을 보완합니다.

**Q: AI가 잘못된 의료 조언을 할 위험은 어떻게 방지하나요?**

**A:** On-Care는 의료 행위가 아닌 '건강 관리 지원'을 목적으로 합니다. 시스템 프롬프트와 출력 필터를 통해 진단, 처방, 투약 관련 답변을 차단하며 , 신뢰할 수 있는 공공 의료 지침 데이터(한국영양학회 등)만을 소스로 사용하는 Closed-domain RAG 방식을 적용합니다.

**Q: 사용자가 앱을 꾸준히 쓰게 만드는 메커니즘이 있나요?**

**A:** 기록의 번거로움을 **Vision AI 자동화**로 낮추고 , RAG를 통해 "내 기록을 아는 맥락 있는 피드백"을 제공해 효능감을 높입니다. 또한 단순 접속이 아닌 실제 저나트륨 식단 달성이나 운동 완료 시에만 보상을 주는 **Streak 시스템**으로 건강 습관 형성을 유도합니다.

**Q: 오프라인 헬스장 연동은 구체적으로 어떻게 이루어지나요?**

**A:** 사용자의 위치를 기반으로 주변 헬스장을 검색해주며 , 사용자가 동의할 경우 앱에 축적된 식단 및 운동 요약 데이터를 트레이너에게 전달하여 초기 상담의 효율성을 극대화합니다.

**Q: 민감한 건강 데이터 보안은 어떻게 처리하나요?**

**A:** 개인 식별 정보(PII)를 제외한 익명화된 벡터 형태로 저장하며, 모든 데이터에 **AES-256 암호화** 및 **TLS 1.3** 전송 보안을 적용합니다. Pinecone 내에서도 user_id 기반 네임스페이스 분리로 데이터 격리를 보장합니다.
