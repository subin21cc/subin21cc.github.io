# 섹션 1. 강의 소개

## 1. 강의 소개

- 스프링 공부 포기하는 이유: 처음부터 IOC, DI, AOP같은 이론 적인 내용부터 학습하기 때문이다. ⇒ “스프링 공부를 왜 해야하는가?”같은 근본적인 질문이 먼저 필요하다.


- 간단한 웹 어리케이션 개발
  - 스프링 프로젝트 생성
  - 스프링 부트로 웹 서버 실행
  - 회원 도메인 개발
  - 웹 MVC 개발
  - DB 연동 - JDBC, JPA, 스프링 데이터 JPA
  - 테스트 케이스 작성
- 프로젝트 사용기술
  - Spring Boot
  - Gradle
  - Thymeleaf
  - JPA (Java Persistence API)
  - HIBERNATE

⇒ 핵심 목표: 스프링홀 웹 어플리케이션을 개발할 때 어떤 기술들이 어떻게 사용되는지 전반적인 감을 잡고 큰 그림을 머릿속에 장작하는 것. 즉, 스프링 학습의 제대로 된 첫 길잡이 역할이다.

- 스프링 기술 그 자체에 매몰 X
- 어떻게 사용해야 하는지에 초점
- 실무에서 거의 사용 하지 않는 오래된/마이너한 스프링 기술 X
