# 5장. 깃허브로 협업하기

## 5-1. 서로 다른 컴퓨터에서 원격 저장소 함께 사용하기

### 원격 저장소 복제하기 - git clone

클론(clone) 또는 클로닝(cloning): 원격 저장소를 지역 저장소로 똑같이 가져오는 것

```bash
git clone 원격 저장소 주소.
```

### 회사 컴퓨터에서 내려받아 작업하기

- 원격 저장소에 새로 올라온 커밋 가져오기

```bash
git pull
```

---

## 5-2. 원격 브랜치 정보 가져오기

### 커밋과 브랜치 이해하기

- `git log --oneline`
  - `git log` : 저장소에 저장된 커밋 히스토리 보여주는 명령어
  - `--oneline` : 커밋을 한 줄로 보여줌

### 원격 브랜치 정보 가져오기 - git fetch

: 실제 내 작업 파일(워킹 디렉토리)을 건드리지 않고, 온라인(GitHub 등)에 새로 올라온 커밋들이 있는지 정보만 가져오는 동작

- git pull 명령은 git fetch 명령과 git merge orgin/main 명령을 합친 것과 같은 기능

---

## 5-3. 협업의 기본 알아보기

### 협업 과정

1. 협엽을 위한 저장소를 만듭니다.
2. 팀원을 추가합니다.
3. 브랜치별로 커밋하고 푸시합니다.
4. 풀 리퀘스트를 요청합니다.
5. 코드 리뷰 & 브랜치 병합

---

## 5-4. 원격 저장소에서 협업하기

1. 협업을 위한 저장소 만들기
2. 공동 작업자 추가하기
3. 프로젝트 관리하기
4. 브랜치 만들기
5. 지역 저장소에서 작업하고 푸시하기
6. 풀 리퀘스트 요청하기
7. 풀 리퀘스트 검토하고 브랜치 병합하기

- 원격 저장소에 없는 브랜치로 푸시하기
```bash
git push -u origin 브랜치명
```
